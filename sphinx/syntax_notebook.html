6. 주피터 문법¶

마크다운(Markdown)은 주피터 노트북(.ipynb)의 마크업 언어이다. 엄밀하게 말해 주피터 노트북은 다음과 같은 내용을 포함하고 있는 JSON 파일이다.

  • 마크다운으로 작성된 내용
  • 마크다운이 아닌 다른 언어로 작성된 내용 (주피터에서 스핑크스 문법으로 작성하기를 보라.)
  • 파이썬 코드와 그 결과
  • 메타데이터

마크다운 문법을 엄격하게 지키지 않은 노트북도 주피터는 제대로 해석한다. 그러나 스핑크스는 융통성을 두지 않고 마크다운 문법에 충실하게 노트북을 해석한다. 따라서 스핑크스를 통해 올바르게 표현된 HTML과 PDF를 얻으려면 이 장이 권고하는 대로 마크다운 문법을 철저히 지켜야 한다.

6.1. 절 제목¶

해시(#) 기호를 이용하여 장과 절의 제목을 나타낸다.

# 장 (chapter): <h1>
## 절 (section): <h2>
### 부절 (subsection): <h3>

스핑크스는 표제들의 수준을 상대적인 것으로 간주한다. 만약 해시가 하나뿐인 표제가 없고 해시가 둘 또는 셋인 표제들만 있다면, ##으로 시작하는 것은 장 수준으로, ###으로 시작하는 것은 절 수준으로 변환된다.

6.2. HTML 코드¶

마크다운은 HTML 코드의 사용을 허용한다. 그러나 HTML뿐만 아니라 PDF도 동시에 만들어야 한다면 HTML 코드를 사용하지 않아야 한다. 주피터가 노트북을 TeX으로 변환할 때, 노트북 안에 쓰인 HTML 코드는 무시되고 텍스트만 남는다. 스핑크스에서도 raw 마크업을 이용하여 HTML 코드를 기입할 수 있다. 그러나 마찬가지로 스핑크스가 reST를 TeX으로 변환할 때, reST 안에 쓰인 HTML 코드는 무시되고 텍스트만 남는다. 반대의 경우에도 그 결과는 같다. reST에서 raw를 이용하여 TeX 매크로를 기입할 수 있지만, HTML로 변환할 때 TeX 매크로들이 무시된다.

6.3. 문자 스타일¶

별표(*) 또는 밑줄(_)을 이용하여 이탤릭체와 볼드체를 표현한다.

  • *italic*: italic
  • _italic_: italic
  • **bold**: bold
  • __bold__: bold

언더스코어는 디렉토리 이름 따위에 사용될 수 있으므로 혼동을 피하기 위해 별표만 사용하는 것이 바람직할 것이다.

볼드체는 사용자 인터페이스에 나타나는 텍스트와 같이 독자의 빠른 식별이 필요한 것에 사용하는 것이 좋다. 이탤릭체는 주로 영어 문서에서 참고문헌을 가리키는 데에 사용된다. 그러나 한국어 문서에서는 기울임체(slanted)의 가독성이 좋지 않기 때문에 기울임체는 사용하지 않는 것이 좋다.

글줄 안에서 타자체(typewriter)를 표현하려면 억음 부호(`)를 이용한다.

`typewriter`

6.4. 이스케이프 문자¶

다음 문자들은 각자 자신의 문법 형식을 충족할 때 마크업으로 인식된다.

\ ` * _ # + ! $ { } [ ] ( ) >

예를 들어, 글줄이 하나 이상의 해시 기호로 시작하고 공백(space)이 뒤따르면 그 글줄은 표제가 된다. #으로 글줄을 시작해야 하는 경우에 그것을 표현하려면 백슬래시()를 앞에 붙여쓰거나 억음 부호(`)로 감싼다.

\#을 해시라고 한다.
`#`을 해시라고 한다.

6.5. 소스 코드¶

만약 백슬래시를 포함하여 입력한 그대로 (literal) 표현하려면 두 개의 억음 부호로 문자열을 감싼다. 이것은 마크업 문자들 때문에 이스케이프 문자를 써야 할 필요가 없음을 의미한다. 이 방법은 프로그래밍 파일의 코드나 명령행(command line)에서의 입출력을 나타내는 데에 사용된다.

``C:\Users\My\AppData``

여러 줄에 걸쳐 코드를 표현하려면 4 자만큼 들여쓰거나 탭을 쓰는 것이다. 주피터는 탭을 4 개의 공백으로 바꾼다.

␣␣␣␣import datetime
␣␣␣␣now = datetime.datetime.now()

또다른 방법은 3개의 억음 부호(```)로 코드를 감싸는 것이다.

```
print(now.year)
print(now.month)
print(now.day)
```

6.6. 목록¶

6.6.1. 비순서 목록¶

별표(*), 더하기(+), 또는 빼기(-) 기호를 비순서 목록을 위한 표지(marker)로 사용할 수 있다. 일관성을 위해 별표만 사용하는 것을 권장하며, 다음 규칙들을 지켜서 작성해야 한다.

  • 목록의 앞과 뒤에 빈 줄을 둔다.
  • 중첩 목록을 만들려면 하위 항목의 별표 앞에 4 개의 공백이나 탭을 둔다.
* 항목
␣␣␣␣* 하위 항목
␣␣␣␣* 하위 항목
* 항목

6.6.2. 순서 목록¶

순서 목록을 만들려면 숫자와 마침표를 사용한다.

1. 첫째
2. 둘째
3. 셋째

번호는 중요하지 않다. 모든 항목에 1.을 써도 무방하다. 순서 목록 안에 비순서 목록을, 또는 그 반대로, 쓰는 것이 가능하다.

6.7. 주의 또는 권고¶

경고문(admonition)을 위한 장치가 마크다운 문법에 없다. 대안은 스핑크스 (reST) 문법을 사용하는 것이다. 주피터는 reST를 비롯한 여러 마크업 언어의 사용을 허용한다. 이에 대한 상세한 설명을 주피터에서 스핑크스 문법으로 작성하기에서 보라.

6.8. 수식¶

글줄 안에서는 $와 $ 사이에 수식을 쓴다. 이것은 $ax^2 + \sqrt{bx} + c = 0$ 이렇게 \(ax^2 + \sqrt{bx} + c = 0\) 표현된다. 여는 $ 뒤와 닫는 $ 앞에 공백을 두지 않도록 주의해야 한다. 그런 수식은 주피터에서 제대로 표현되지만, 스핑크스는 수식 명령어 앞에 :raw-latex:을 삽입하여, 결과적으로 올바르게 표현되지 않는다.

여러 줄에 걸쳐 수식을 쓰려면 $$로 수식을 감싼다.

$$
\frac{1}{\Bigl(\sqrt{\phi \sqrt{5}}-\phi\Bigr) e^{\frac25 \pi}} =
1+\frac{e^{-2\pi}} {1+\frac{e^{-4\pi}} {1+\frac{e^{-6\pi}}
{1+\frac{e^{-8\pi}} {1+\ldots} } } }
$$
\[\frac{1}{\Bigl(\sqrt{\phi \sqrt{5}}-\phi\Bigr) e^{\frac25 \pi}} = 1+\frac{e^{-2\pi}} {1+\frac{e^{-4\pi}} {1+\frac{e^{-6\pi}} {1+\frac{e^{-8\pi}} {1+\ldots} } } }\]

코드 셀(Code cell)에서도, 아래 예처럼, 수식을 표현하는 것이 가능하다.

display(Math(r'F(k) = \int_{-\infty}^{\infty} f(x) e^{2\pi i k} dx'))

수식 안에서 문자들을 정자체(upright)로 표현하려면 \mathrm을 사용한다.

MSC.NASTRAN$^{\mathrm{TM}}$: MSC.NASTRAN\(^{\mathrm{TM}}\)

6.9. 상호참조¶

다음의 형식을 이용하여 상호참조를 만든다.

[text](#heading-title)
[text](foo.ipynb)
[text](foo.ipynb#heading-title)
주피터에서 제목을 만드는 방법에 대해 [절 제목](#절-제목)을 보라.

괄호 안에 절 제목을 쓸 때 공백이 있으면 그 대신 하이픈(-)을 사용한다. 다른 파일에 있는 절을 참조하려면 # 앞에 원본 파일의 이름을 쓴다. 주피터에서 제목을 만드는 방법에 대해 절 제목을 보라.

주피터 노트북에서 reST 파일로 연결하는 것은 가능하지만 reST 파일 안의 특정 절로 연결하는 것은 가능하지 않다. 반대로 reST 파일에서 노트북 파일로 연결하는 것 역시 가능하지 않다.

6.10. 그림¶

이미지 파일을 삽입하는 방법은 상호참조를 만드는 것과 비슷하다.

![텍스트](path/foo.png)

[텍스트]는 스핑크스에 의해 HTML과 TeX에서 그림 제목(caption)으로 바뀐다. 그러나 그림 제목에 대한 상호참조는 만들 수 없기 때문에, 반드시 필요한 경우에만 부연 설명으로서 사용하는 것이 바람직하다.

코드 셀에서 그래프를 생성하는 프로그램을 작성하는 경우에 이미지 포맷을 SVG로 지정할 수 있다.

%matplotlib inline
%config InlineBackend.figure_format = 'svg'

In [2]:

%matplotlib inline
%config InlineBackend.figure_format = 'svg'
import numpy as np
import matplotlib.pyplot as plt
from math import pi, asin
theta = np.linspace(0, 4.*pi, 100)
fig, ax = plt.subplots(figsize = (5, 5))
for rho in [0., 0.4, 0.8, 0.9, 1.]:
    phase = asin(rho)
    ax.plot(np.cos(theta), np.sin(theta + phase), label = "$\\rho = %.1f$" % rho)
ax.legend(loc = 4);

6.11. 표¶

표를 만드는 규칙은 다음과 같다.

  • 각 칼럼을 세로줄(|)로 구분한다.
  • 첫 줄 아래에 빼기(-) 기호를 이용하여 구분선을 긋는다.
**First header** | **Second header** | **Third header**
-----------------|-------------------|-----------------
First            | Second            | Third
Forth            | Fifth             | Sixth
First header Second header Third header
First Second Third
Forth Fifth Sixth
