1 변수와 함수
텍 문법에서 변수와 함수는 형태적으로 서로 구분되지 않는다. 그러나 expl3에서는 뚜렷하게 구분된다.
함수는 콜론(:)을 포함하고 변수는 그렇지 않다.
\l_tmpa_tl % 변수
\tl_set:Nn % 함수
콜론과 콜론 뒤의 글자들까지 함수 이름의 일부이다. 콜론 뒤의 글자들이 그 함수가 무엇을 요구하는지
암시한다.
변수의 끝에 오는 글자들이 그 변수의 유형을 암시한다.
\l_tmpa_tl % 토큰 리스트
\l_tmpa_int % 정수
\g_tmpb_dim % 길이
\g_tmpb_clist % 콤마 리스트
변수의 처음에 오는 글자들이 전역
(global)
변수인지, 지역
(local)
변수인지 나타낸다. tmpa 또는 tmpb를
포함하는 변수들은 미리 정의되어 있으며 사용자들은 그것들을 자유롭게 사용할 수 있다.
함수 인자
\tl_set:Nn \l_tmpa_tl { Hello }
\int_set:Nn \l_tmpa_int { 27 }
\l_tmpa_tl \space
\int_use:N \l_tmpa_int
Hello 27
토큰 리스트를 처리하는 모든 함수들은 \tl_로 시작하고, 정수를 처리하는 모든 함수들은 \int_로 시
작한다. 콜론 뒤에 N은 그 함수가 단일 토큰을 요구함을 의미한다. 대부분의 경우 그것은 하나의 콘트롤
1
시퀀스, 즉 매크로이다. 콜론 뒤에 오는 n은 그 함수가 중괄호로 싸인 토큰 리스트, 즉 문자열이다.
1
따라서
Nn은 두 개의 인자를 요구하는데 처음 것은 명령을, 다음 것은 문자열이어야 한다는 것을 나타낸다.
지금까지 약술한 expl3의 명명 규약을 지키지 않고 변수와 함수를 만들었을 때 별 문제가 생기지 않는다.
그 규약들이 낯설고 번잡해 보여도 익숙해지면 코드를 이해하고 수정하기가 용이해진다.
밑줄과 공백
expl3의 변수 및 함수의 이름은 “기호를 매크로 이름으로 사용할 수 없다”는 기본적인 규칙에서 벗어난다.
게다가 밑줄
( _ )
은 수식 모드에서만 사용할 수 있다. 이를 가능하게 하는 것이 \ExplSyntaxOn 명령이다.
\ExplSyntaxOn
\tl_new:N \l_foo_tl
\cs_new:Nn \foo_goo: { ... }
\ExplSyntaxOff
\ExplSyntaxOn과 \ExplSyntaxOff 사이에서 모든 공백이 무시된다. 공백이 필요하면 틸데
(~)
나
\space 같은 공백 명령을 써야 한다.
변수 만들기
\def나 \newcommand와 달리, expl3에서는 새로운 변수를 먼저 만들고 그 다음에 값을 대입해야 한다.
\tl_new:N \l_foo_tl
\tl_set:Nn \l_foo_tl { Hello }
전역 변수와 지역 변수
\tl_set:Nn \l_tmpa_tl { A }
\group_begin:
\tl_set:Nn \l_tmpa_tl { B } \l_tmpa_tl
\group_end:
\l_tmpa_tl
\group_begin:
\tl_gset:Nn \l_tmpa_tl { C }
\group_end:
\l_tmpa_tl
BAC
\group_begin:과 \group_end: 사이에서 설정된 변수는 그 안에서만 유효하다. 그 범위를 벗어나서도
유효하게 하려면 \tl_gset:Nn 함수를 사용해야 한다. 다른 변수 유형에도 같은 함수가, 이를테면 정수형
변수를 전역적으로 변경하는 \int_gset:Nn 함수가 있다.
함수 만들기
expl3로 만들어진 함수들을 사용자 인터페이스 수준에서, 즉 \begin{document}와 \end{document}
사이에서 사용할 수 없다. \NewDocumentCommand와 \NewDocumentEnvironment에 의해 만들어진
1
엄밀히 말해 토큰 리스트는 처리되지 않은 문자들의 집합이고, string 변수 유형과 구분되어야 한다. 그러나 이 글에서는 별다른
언급이 없는 한 매크로가 아닌 인자를 의미하는 말로서 ‘문자열’이 사용될 것이다.
2
명령들이 expl3 함수들을 호출할 수 있다.
\NewDocumentCommand \nova { m }
{
\nova_iterate:n { #1 }
}
\cs_new:Npn \nova_iterate:n #1
{
\int_step_inline:nnnn {1}{1}{#1}{ ★ }
}
\nova{5}
★★★★★
\cs_new:Npn는 세 가지 인자를 요구한다.
• 함수 이름
• 패러미터 (#1, #2, 󽴿)
• 코드 (함수 정의)
\nova_iterate:n이문자열 인자를취하도록정의되었다.인자가 필요없는 함수를만들려면\cs_new:Nn을
사용하라.
2 옵션 키
함수의 작동을 쉽게 제어하기 위한 목적으로 키와 값의 쌍으로 이루어진 옵션 세트가 사용된다.
\keys_define:nn { nova }
{
size .tl_set:N = \l_nova_font_size_tl,
color .tl_set:N = \l_nova_color_tl
}
\DeclareDocumentCommand \NovaSetup { m }
{
\keys_set:nn { nova }{ #1 }
}
\DeclareDocumentCommand \nova { o m }
{
\group_begin:
\NovaSetup{#1}
\int_step_inline:nnnn {1}{1}{#2}
{
\l_nova_font_size_tl
\textcolor{\l_nova_color_tl}{★}
}
\group_end:
}
\nova[color=red,size=\LARGE]{3}
★★★
이 예에서 size 키에 대응하는 \l_nova_font_size_tl 변수가 정의되었다. 여기에서 사용된 함수
.\tl_set:N가 백슬래시가 아닌 마침표로 시작한다는 것을 유념하라.
\nova에 앞서 \NovaSetup에 의해 설정된 옵션은 전역적으로 유효하다.
3
\NovaSetup{color=blue} % 전역 설정
\nova[color=yellow]{3} % 지역 설정
