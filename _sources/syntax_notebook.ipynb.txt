    "# 주피터 문법\n",
    "\n",
    "마크다운(Markdown)은 주피터 노트북(.ipynb)의 마크업 언어이다. 엄밀하게 말해 주피터 노트북은 다음과 같은 내용을 포함하고 있는 JSON 파일이다.\n",
    "\n",
    "* 마크다운으로 작성된 내용\n",
    "* 마크다운이 아닌 다른 언어로 작성된 내용 ([주피터에서 스핑크스 문법으로 작성하기](reST_in_nb.ipynb)를 보라.)\n",
    "* 파이썬 코드와 그 결과\n",
    "* 메타데이터 \n",
    "\n",
    "마크다운 문법을 엄격하게 지키지 않은 노트북도 주피터는 제대로 해석한다. 그러나 스핑크스는 융통성을 두지 않고 마크다운 문법에 충실하게 노트북을 해석한다. 따라서 스핑크스를 통해 올바르게 표현된 HTML과 PDF를 얻으려면 이 장이 권고하는 대로 마크다운 문법을 철저히 지켜야 한다."

    "## 절 제목\n",
    "\n",
    "해시(#) 기호를 이용하여 장과 절의 제목을 나타낸다.\n",
    "\n",
    "```\n",
    "# 장 (chapter): <h1>\n",
    "## 절 (section): <h2>\n",
    "### 부절 (subsection): <h3>\n",
    "```\n",
    "\n",
    "스핑크스는 표제들의 수준을 상대적인 것으로 간주한다. 만약 해시가 하나뿐인 표제가 없고 해시가 둘 또는 셋인 표제들만 있다면, `##`으로 시작하는 것은 장 수준으로, `###`으로 시작하는 것은 절 수준으로 변환된다."

    "\n",
    "마크다운은 HTML 코드의 사용을 허용한다. 그러나 HTML뿐만 아니라 PDF도 동시에 만들어야 한다면 HTML 코드를 사용하지 않아야 한다. 주피터가 노트북을 TeX으로 변환할 때, 노트북 안에 쓰인 HTML 코드는 무시되고 텍스트만 남는다. 스핑크스에서도 `raw` 마크업을 이용하여 HTML 코드를 기입할 수 있다. 그러나 마찬가지로 스핑크스가 reST를 TeX으로 변환할 때, reST 안에 쓰인 HTML 코드는 무시되고 텍스트만 남는다. 반대의 경우에도 그 결과는 같다. reST에서 `raw`를 이용하여 TeX 매크로를 기입할 수 있지만, HTML로 변환할 때 TeX 매크로들이 무시된다.     "## 문자 스타일\n",
    "\n",
    "별표(*) 또는 밑줄(_)을 이용하여 이탤릭체와 볼드체를 표현한다.\n",
    "\n",
    "* `*italic*`: *italic*\n",
    "* `_italic_`: _italic_\n",
    "* `**bold**`: **bold**\n",
    "* `__bold__`: __bold__\n",
    "\n",
    "언더스코어는 디렉토리 이름 따위에 사용될 수 있으므로 혼동을 피하기 위해 별표만 사용하는 것이 바람직할 것이다.\n",
    "\n",
    "**볼드체**는 사용자 인터페이스에 나타나는 텍스트와 같이 독자의 빠른 식별이 필요한 것에 사용하는 것이 좋다.\n",

    "\n",
    "별표(*), 더하기(+), 또는 빼기(-) 기호를 비순서 목록을 위한 표지(marker)로 사용할 수 있다. 일관성을 위해 별표만 사용하는 것을 권장하며, 다음 규칙들을 지켜서 작성해야 한다.\n",

    "* 항목\n",

    "## 수식\n",
    "\n",
    "글줄 안에서는 \\$와 \\$ 사이에 수식을 쓴다. 이것은 `$ax^2 + \\sqrt{bx} + c = 0$` 이렇게 $ax^2 + \\sqrt{bx} + c = 0$ 표현된다.\n",
    "여는 `$` 뒤와 닫는 `$` 앞에 공백을 두지 않도록 주의해야 한다. 그런 수식은 주피터에서 제대로 표현되지만, 스핑크스는 수식 명령어 앞에 `:raw-latex:`을 삽입하여, 결과적으로 올바르게 표현되지 않는다.\n",
    "\n",
    "여러 줄에 걸쳐 수식을 쓰려면 `$$`로 수식을 감싼다.\n",
    "\n",
    "```\n",
    "$$\n",
    "\\frac{1}{\\Bigl(\\sqrt{\\phi \\sqrt{5}}-\\phi\\Bigr) e^{\\frac25 \\pi}} =\n",
    "1+\\frac{e^{-2\\pi}} {1+\\frac{e^{-4\\pi}} {1+\\frac{e^{-6\\pi}}\n",
    "{1+\\frac{e^{-8\\pi}} {1+\\ldots} } } }\n",
    "$$\n",
    "```\n",
    "\n",
    "$$\n",
    "\\frac{1}{\\Bigl(\\sqrt{\\phi \\sqrt{5}}-\\phi\\Bigr) e^{\\frac25 \\pi}} =\n",
    "1+\\frac{e^{-2\\pi}} {1+\\frac{e^{-4\\pi}} {1+\\frac{e^{-6\\pi}}\n",
    "{1+\\frac{e^{-8\\pi}} {1+\\ldots} } } }\n",
    "$$\n",
    "\n",
    "코드 셀(Code cell)에서도, 아래 예처럼, 수식을 표현하는 것이 가능하다.\n",
    "\n",
    "```\n",
    "display(Math(r'F(k) = \\int_{-\\infty}^{\\infty} f(x) e^{2\\pi i k} dx'))\n",
    "```\n",
    "\n",
    "수식 안에서 문자들을 정자체(upright)로 표현하려면 `\\mathrm`을 사용한다.\n",
    "\n",
    "`MSC.NASTRAN$^{\\mathrm{TM}}$`: MSC.NASTRAN$^{\\mathrm{TM}}$\n"

    "## 상호참조\n",
    "\n",
    "다음의 형식을 이용하여 상호참조를 만든다.\n",
    "\n",
    "```\n",
    "[text](#heading-title)\n",
    "[text](foo.ipynb)\n",
    "[text](foo.ipynb#heading-title)\n",
    "주피터에서 제목을 만드는 방법에 대해 [절 제목](#절-제목)을 보라.\n",
    "```\n",
    "\n",
    "괄호 안에 절 제목을 쓸 때 공백이 있으면 그 대신 하이픈(`-`)을 사용한다.\n",
    "다른 파일에 있는 절을 참조하려면 `#` 앞에 원본 파일의 이름을 쓴다.\n",
    "주피터에서 제목을 만드는 방법에 대해 [절 제목](#절-제목)을 보라.\n",
    "\n",
    "주피터 노트북에서 reST 파일로 연결하는 것은 가능하지만 reST 파일 안의 특정 절로 연결하는 것은 가능하지 않다.\n",
