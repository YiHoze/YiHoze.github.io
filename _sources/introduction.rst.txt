.. _introduction:

*****
소개
*****

스핑크스(Sphinx)와 주피터(Jupyter)는 파이썬에 기반한 문서화 소프트웨어이다. 이 문서는 스핑크스와 주피터를 활용하여 HTML 및 PDF 문서를 동시에 생성하는 데에 요구되는 다음과 같은 것들을 설명한다.

* 스핑크스와 주피터 문법
* HTML 템플릿을 변경하는 방법
* 레이텍 (LaTeX) 템플릿과 스타일을 변경하는 방법

.. _DirectoryTree:

문서 디렉토리 구조
==================

이 문서가 포함된 디렉토리의 구조는 다음과 같다.

::

  D:
  +--- docs
       +--- _static
       +--- _templates
       +--- bin
            +--- python
       +--- example
       +--- howto
       +--- locale
       +--- manual
       +--- modules
       +--- tutorial

참고로, 이전 판은 다음과 같았다. 

::

  D:
  +--- pyFlutter       
       +--- docs
       |    +--- _static
       |    +--- _templates
       |    +--- bin
       |    |    +--- applications
       |    |    +--- fonts
       |    +--- howto
       |    +--- locale
       |    +--- manual
       |    +--- modules
       |    +--- tutorial
       +--- MathJax
       +--- Nastran
       +--- pyFlutter
       +--- Universal

루트 디렉토리의 위치가 바뀌더라도 이 디렉토리 구조는 유지되어야 한다. docs 디렉토리 아래에 이 문서를 비롯한 문서 파일들과 문서화에 필요한 갖가지 설정 파일들과 실행 파일들이 들어 있다.
레이텍 템플릿을 비롯한 문서 생성에 필요한 보조 파일들이 다음과 같이 다섯 디렉토리에 들어 있다.

* _static: HTML 스타일
* _template: 레이텍 템플릿
* bin: 문서 생성을 돕기 위한 스크립트와 레이텍 스타일 (이전 판에는 레이텍 스타일 파일들이 _template 디렉토리에 있었다.)
* locale: 한국어 로캘
* modules: 변경된 스핑크스 모듈

이전 판이 제공한 응용 소프트웨어와 폰트를 이용해야 한다. 그것들은 이전 판에서 bin 디렉토리에 들어있었다.
:doc:`installation`\ 를 보라.

나머지 디렉토리들이 문서 파일들을 담고 있다.

* example: 보고서 예
* howto: 스핑크스와 주피터를 이용하여 HTML과 PDF 문서를 만드는 방법에 대한 설명과 지침 (이 문서)

용어
=====

이 문서는 편의를 위해 directive, role, markup을 엄격하게 구분하여 사용하지 않고 모두 "마크업"이라 통칭한다.
같은 이유로 이 문서에서 "주피터 문법"은 Markdown을, "스핑크스 문법"은 reStructuredText를 가리킨다.
